# Color and Theme

Color communicates meaning.

It must not be the only way to communicate state.

## Theme Model

Light and Dark use the same semantic tokens.

Values change, meaning does not.

## Required Semantic Colors

- `surface-page`
- `surface-card`
- `surface-subtle`
- `text-primary`
- `text-muted`
- `text-subtle`
- `border-default`
- `border-strong`
- `action-primary`
- `action-secondary`
- `action-danger`
- `state-success`
- `state-warning`
- `state-error`
- `focus-ring`

## Brand Public

Brand Public should be:

- White based.
- Calm.
- Modern.
- Content first.

Accent color should guide attention, not dominate the page.

## Creator Public

Creator Public can have stronger personality through tokens.

Chikage direction:

- Modern Japanese calm.
- Paper.
- Ink.
- Indigo.
- Seasonal feeling.

Avoid:

- Shrine fantasy.
- Period drama.
- Generic Japanese template.

## Studio / Admin

Studio colors must support long work sessions.

They should:

- Reduce glare.
- Make states obvious.
- Keep danger separate.
- Keep forms readable.

## Contrast

All text and interactive controls should meet WCAG AA-level contrast.

Primary action text must be readable in Light, Dark, hover, focus, active, and
disabled states.
