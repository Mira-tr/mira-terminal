# Toast and Feedback

## Purpose

Toast gives short feedback.

Important state must also remain visible in the screen.

## Variants

- Success.
- Warning.
- Error.
- Info.
- Loading.

## Required Feedback

- Saving.
- Saved.
- Unsaved.
- Validation Error.
- Export waiting.
- Build running.
- Build success.
- Build failure.
- Publish ready.
- Backup complete.
- Import preview.
- Rollback complete.

## Keyboard

Toast should not steal focus.

Actionable toast controls must be keyboard reachable.

## Accessibility

- Use aria-live.
- Do not auto-dismiss critical errors too quickly.

## Mobile Behavior

- Does not cover primary action for long.
- Wraps text.

## Studio

Use toast for temporary acknowledgement.

Use inline panels for important state.
