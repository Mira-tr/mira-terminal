# Badge and Tag

## Purpose

Badge shows status.

Tag shows category or user-facing classification.

## Variants

Badge:

- Success.
- Warning.
- Error.
- Info.
- Draft.
- Public.

Tag:

- Category.
- Filter.
- Selected filter.

## States

- Default.
- Selected.
- Focus-visible when interactive.
- Disabled.

## Keyboard

Interactive tags must be buttons or checkboxes.

Static tags must not look clickable.

## Accessibility

Do not use color alone.

Include text.

## Mobile Behavior

Tags wrap.

Long tags should not overflow.

## Brand Public

Use tags for editorial classification.

## Creator Public

Use tags for activity and works.

## Studio

Use badges for status and tags for filters.
