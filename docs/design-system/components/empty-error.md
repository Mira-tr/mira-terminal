# Empty State and Error Panel

## Purpose

Empty state explains why nothing is shown.

Error panel explains what broke and what to do next.

## Variants

- Empty.
- Filtered empty.
- Not configured.
- Error.
- Warning.
- Recovery.

## States

- Empty.
- Loading.
- Error.
- Retry available.

## Keyboard

Primary next action must be reachable.

## Accessibility

- Use heading and text.
- Do not rely only on icon.

## Mobile Behavior

- Keep short.
- CTA wraps.

## Brand Public

Empty state should feel curated.

## Creator Public

Empty state should explain creator context.

## Studio

Empty state should explain the next setup action.
