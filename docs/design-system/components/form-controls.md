# Form Controls

## Purpose

Form controls collect data safely.

## Components

- Input.
- Textarea.
- Select.
- Checkbox.
- Radio.
- Toggle.

## States

- Default.
- Hover.
- Focus-visible.
- Disabled.
- Read-only.
- Required.
- Error.
- Success.
- Loading.

## Keyboard

- Labels must focus controls.
- Tab order follows visual order.
- Space toggles checkbox/radio/toggle.
- Arrow keys work for radio groups when applicable.

## Accessibility

- Every control has a label.
- Error message is near the field.
- Required reason is understandable.
- Help can open when needed.

## Beginner Rules

- Do not show internal IDs.
- Do not ask for storage path.
- Safe default values.
- One group of fields at a time in wizards.

## Mobile Behavior

- Full-width controls.
- Large enough tap area.
- Avoid tiny side-by-side fields.

## Brand Public

Use minimal forms.

## Creator Public

Use forms only when needed.

## Studio

Use compact but readable production forms.
