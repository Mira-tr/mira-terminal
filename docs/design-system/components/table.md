# Table

## Purpose

Table shows structured repeated data.

Use in Studio when scanning and comparison matter.

## Variants

- Data table.
- Status table.
- Import preview table.
- Diagnostics table.

## States

- Default.
- Hover row.
- Selected row.
- Focus-visible controls.
- Empty.
- Error.
- Loading.

## Keyboard

Interactive cells must contain real controls.

Tab order should not become excessive.

## Accessibility

- Use table semantics for tabular data.
- Headers must describe columns.

## Mobile Behavior

Options:

- Horizontal scroll with visible affordance.
- Stack rows into cards.
- Hide non-critical columns.

## Brand Public

Avoid tables unless truly needed.

## Creator Public

Avoid tables unless content is tabular.

## Studio

Tables are appropriate for diagnostics, logs, imports, and lists.
