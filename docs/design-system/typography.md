# Typography

Typography must support long Japanese reading.

It must not be only decorative.

## Text Roles

| Role | Purpose |
| --- | --- |
| Display | One strong message, usually on Home or major hero. |
| Page title | Main title for a page. |
| Section title | Starts a content section. |
| Card title | Names one item. |
| Body | Main reading text. |
| Caption | Small support text. |
| Label | Form or UI label. |
| Code | Technical value or path in Advanced mode. |
| Numeric status | Counts, health values, build results. |

## Japanese Reading

Long Japanese text should use:

- Comfortable line-height.
- Moderate line length.
- Clear paragraph spacing.
- Avoid tiny body text.

Recommended reading max width:

- Public prose: about 34 to 42 Japanese characters per line.
- Studio help text: shorter and scannable.

## Mobile Rule

Large display text must never overlap body text on mobile.

If the display text cannot fit:

- Reduce size.
- Wrap naturally.
- Shorten the copy.
- Move decorative text away from content.

Do not let giant type cover buttons, cards, or form fields.

## Numeric Status

Numbers in Studio should be easy to scan.

Use tabular or stable-width numeric presentation where possible.

Examples:

- Validation count.
- Backup age.
- Build result.
- Public JSON count.

## Code Text

Use code style for:

- File paths.
- Schema fields.
- IDs.
- Commands.

Beginner mode should avoid exposing code text unless needed.
