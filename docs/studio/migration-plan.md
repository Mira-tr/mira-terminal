# Studio Migration Plan

## Phase 0

- Architecture and technology decision.
- Tauri foundation.
- Project root validation.
- Project status display.
- Public JSON read.
- Safe Write PoC for one target.
- Build PoC with fixed command.
- Git read-only status.
- Security and rollback docs.

## Phase 1

- Public JSON automatic placement.
- Backup and validation integration.
- Preview workflow.
- File watch for canonical JSON changes.

## Phase 2

- Editor integration.
- localStorage migration strategy.
- Module mapping in Studio UI.
- Browser Admin compatibility bridge.

## Phase 3

- Build integration.
- Activity Log integration.
- Publish Preflight parity with Browser Admin.

## Phase 4

- Git commit assistance.
- Push assistance.
- GitHub Pages status check.

## Phase 5

- Auto update.
- Plugin model.
- Multi Creator permissions.
- Role-based workspace access.

Browser Admin remains available until Studio can complete the full edit, validate, write, build, preview, backup, and release-prep cycle without regressions.
