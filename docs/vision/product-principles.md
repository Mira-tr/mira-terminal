# Product Principles

RELMUA is a Brand OS.

It is not only a website, admin panel, static generator, or desktop tool.

## Operation

RELMUA must support daily operation:

- What changed?
- What is broken?
- What is ready?
- What needs backup?
- What can be published?
- What should be done next?

The product should make these visible without forcing users to inspect files.

## UX

RELMUA should feel calm and safe.

The user should not feel that one wrong click can destroy the project.

The user should see:

- One next action.
- Clear save state.
- Clear validation.
- Clear recovery path.
- Human explanations.

## Maintainability

The system should remain understandable after years of growth.

This requires:

- Data contracts.
- Registries.
- Validation.
- Backup.
- Migration.
- Decision records.
- Clear module boundaries.

## Future

RELMUA must be able to grow into:

- Desktop Studio.
- Browser Admin.
- Public Site.
- Creator Sites.
- Future PWA.
- Future plugins.
- Future modules.

Growth must not require replacing the core idea.

## Brand OS Rule

Every feature must support brand operation.

If a feature only makes one screen convenient but weakens data, recovery,
validation, or long-term clarity, it does not belong yet.
